# Flip-over-templates — WS4 Blok 2

Twee templates: één voor blok 2a (3×3 matrix "Drie stemmen × drie aannames"), één voor blok 2b (toetsbare hypothese-lijst). Beide kunnen door de docent live op flip-over getekend worden, of vooraf voorbereid op A1-flips als hand-outs.

---

## Template 1 — Blok 2a: "Drie stemmen × drie aannames"

Eén flip-over A1 per duo. Tekening op de flip-over:

```
┌──────────────────┬─────────────────┬─────────────────┬─────────────────┐
│                  │ AANNAME 1:      │ AANNAME 2:      │ AANNAME 3:      │
│  ROL ↓           │ [korte tag]     │ [korte tag]     │ [korte tag]     │
├──────────────────┼─────────────────┼─────────────────┼─────────────────┤
│                  │                 │                 │                 │
│  KAART 1         │                 │                 │                 │
│  Skeptische peer │                 │                 │                 │
│                  │                 │                 │                 │
│                  │                 │                 │                 │
├──────────────────┼─────────────────┼─────────────────┼─────────────────┤
│                  │                 │                 │                 │
│  KAART 2         │                 │                 │                 │
│  Domein-expert   │                 │                 │                 │
│                  │                 │                 │                 │
│                  │                 │                 │                 │
├──────────────────┼─────────────────┼─────────────────┼─────────────────┤
│                  │                 │                 │                 │
│  KAART 3         │                 │                 │                 │
│  Ethicus         │                 │                 │                 │
│                  │                 │                 │                 │
│                  │                 │                 │                 │
└──────────────────┴─────────────────┴─────────────────┴─────────────────┘
```

### Invul-instructie voor studenten (op een handout of op de eerste regel van de flip-over)

> **Per cel: schrijf de KERN in 1–2 zinnen.** Niet de output kopiëren-plakken — interpreteren. Wat is het belangrijkste argument dat deze rol op deze aanname leverde?
>
> **Markeer met een ✱** de cel waar de tegenspraak het sterkst was — dat is je kandidaat-aanname voor blok 2b.

### Voorbeeld-invulling (test-aanname *"medewerkers tonen weerstand omdat ze digitaal niet vaardig zijn"*)

| ROL ↓ | AANNAME 1: digitale weerstand | AANNAME 2: ... | AANNAME 3: ... |
|-------|-------------------------------|----------------|----------------|
| **Skeptische peer** | Veronderstelt dat weerstand = vaardigheidstekort. Faalt als bezwaar in autonomie zit. ✱ | ... | ... |
| **Domein-expert** | Caluwé kleur-denken: kan ook 'geel' (machts/belangen) of 'rood' (HRM) zijn, niet alleen 'blauw' (causaal vaardigheid → training). | ... | ... |
| **Ethicus** | Reduceert structureel issue (rol-uitholling door digitalisering) tot individueel tekort. Stakeholder: medewerkers >50 met expertise-positie. | ... | ... |

De ✱-markering wijst naar aanname 1 (rij Skeptische peer) als input voor blok 2b.

## Template 2 — Blok 2b: "Toetsbare hypothese-lijst"

Eén flip-over A1 per duo (kan tweede vel zijn, of andere helft van flip 1).

```
┌──────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  GEKOZEN AANNAME (✱ uit blok 2a):                                   │
│                                                                      │
│  ___________________________________________________________________  │
│                                                                      │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  OM DEZE AANNAME OVEREIND TE HOUDEN IN HET VELD MOET GELDEN DAT...   │
│                                                                      │
│  1. _______________________________________________________________  │
│     → Te toetsen via: ____________________________________________   │
│                                                                      │
│  2. _______________________________________________________________  │
│     → Te toetsen via: ____________________________________________   │
│                                                                      │
│  3. _______________________________________________________________  │
│     → Te toetsen via: ____________________________________________   │
│                                                                      │
│  4. _______________________________________________________________  │
│     → Te toetsen via: ____________________________________________   │
│                                                                      │
│  5. _______________________________________________________________  │
│     → Te toetsen via: ____________________________________________   │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

### Invul-instructie voor studenten

> **Vul minstens 3 condities in, idealiter 5.** Elke conditie is **observeerbaar of meetbaar** — niet "er moet draagvlak zijn" (te vaag) maar bijvoorbeeld "minimaal 70% van uitvoerend medewerkers slaagt voor een korte digitale taaktest binnen 4 weken".
>
> **Per conditie: één concrete veld-methode.** Interview-vraag, observatie-criterium, document-bron, enquête-item, of korte test.

### Kwaliteitscriteria voor de hypothese-lijst

| ✓ Goed | ✗ Niet goed |
|--------|-------------|
| Conditie heeft een **getal of verifieerbare uitspraak** ("70% slaagt voor", "<10% noemt vaardigheidsgebrek"). | Conditie is een buzzword ("draagvlak", "betrokkenheid", "transparantie"). |
| Methode is **concreet** (interview-vraag letterlijk, observatie-protocol). | Methode is een vakgebied ("kwalitatief onderzoek", "interviews"). |
| Stakeholder/doelgroep is **specifiek** ("uitvoerend medewerkers in functie X"). | Stakeholder is generiek ("de medewerkers"). |
| Tijdsindicatie aanwezig ("binnen 4 weken", "in afgelopen 2 jaar"). | Geen tijd-anker. |

### Voorbeeld-invulling (test-aanname *"medewerkers tonen weerstand omdat ze digitaal niet vaardig zijn"*)

```
GEKOZEN AANNAME: Medewerkers tonen weerstand omdat ze digitaal 
                 niet vaardig zijn (uit ✱-cel)

OM DEZE AANNAME OVEREIND TE HOUDEN IN HET VELD MOET GELDEN DAT...

1. ≥70% van uitvoerend medewerkers in afdeling X slaagt voor een 
   korte digitale taaktest binnen 4 weken zonder hulp.
   → Te toetsen via: praktijktest met 8–10 willekeurig gekozen 
     medewerkers (intern, vrijwillig).

2. Bij niet-slagende groep wordt 'vaardigheidsgebrek' in ≥60% van 
   de open interviews genoemd als hoofdoorzaak (boven autonomie, 
   werkdruk, of richting van de organisatie).
   → Te toetsen via: semi-gestructureerd interview met de niet-
     slagende groep, open beginvraag "wat zou je doen meedoen?".

3. Eerdere trainingen op vergelijkbare digitale tools hebben in 
   deze afdeling >60% gedragsverandering opgeleverd.
   → Te toetsen via: document-analyse trainingsregister + 
     beoordelings-/voortgangsrapporten afgelopen 2 jaar.

4. Er is geen recente herstructurering of taakverschuiving die 
   weerstand kan verklaren los van digitale vaardigheid.
   → Te toetsen via: interview met afdelingshoofd + HR-directeur 
     over organisatieveranderingen afgelopen 18 maanden.

5. De digitale tools zelf hebben een bruikbaarheid die gangbare 
   ontwerpnormen haalt (geen UX-tekort).
   → Te toetsen via: korte usability-test met 5 medewerkers + 
     kort UX-criterium-checklist (System Usability Scale).
```

Deze lijst gaat als input naar blok 3 (A3-update) en uiteindelijk naar PvA H4.

## Praktische tip voor de docent

Bij **online of hybride uitvoering**: vervang flip-over door een digitale Miro-board of FigJam met dezelfde matrix als template. Studenten plakken er post-its in. Werkt goed bij groepen waar fysiek samenkomen niet kan, maar **de papier-eerst-discipline is dan moeilijker te bewaken** — schermfocus stijgt.

Bij **echte fysieke flip-overs**: zorg dat elk duo aan een eigen tafel met staande flip-over zit. Laptops moeten *naast* de flip-over passen, niet erbovenop. Studenten typen aan de zijkant, schrijven op de flip in het midden.
